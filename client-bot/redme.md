# План разработки Telegram-бота на Python (Aiogram 3)

Этот документ описывает пошаговый план создания современного, асинхронного Telegram-бота для проекта, используя библиотеку `aiogram` (версия 3.x).

## 1. Подготовка и настройка окружения
* **Регистрация бота:** Создать нового бота через [@BotFather](https://t.me/BotFather) в Telegram и получить API Token.
* **Инициализация проекта:**
  * Создать виртуальное окружение: `uv venv`
  * Активировать его: `source .venv/bin/activate`
  * Установить базовые библиотеки: 
    ```bash
    uv pip install aiogram pydantic-settings aiohttp
    ```
* **Переменные окружения:** Создать файл `.env` для хранения конфиденциальных данных (токен бота, URL вашего API).

## 2. Архитектура и структура проекта
Для обеспечения масштабируемости рекомендуется использовать следующую структуру:

```text
client-bot/
├── .env                  # Настройки окружения (токены, URL)
├── config.py             # Скрипт для загрузки переменных из .env (через Pydantic)
├── main.py               # Точка входа, запуск бота (polling или webhook)
├── handlers/             # Обработчики команд и сообщений
│   ├── __init__.py
│   ├── start.py          # Обработка /start и /help
│   └── user_actions.py   # Специфичные команды пользователя
├── keyboards/            # Файлы с Inline и Reply клавиатурами
│   ├── __init__.py
│   └── main_menu.py
├── api_client/           # Взаимодействие с вашим основным Backend API (Auth, Users)
│   ├── __init__.py
│   └── client.py         # Асинхронные HTTP запросы (aiohttp или httpx)
└── requirements.txt      # Список зависимостей
```

## 3. Разработка базового функционала
* **Настройка конфигурации (`config.py`):** Настроить чтение токена и URL бэкенда с помощью `pydantic-settings`.
* **Инициализация бота (`main.py`):** Создать экземпляры `Bot` и `Dispatcher`. Настроить базовое логирование.
* **Базовые команды (`handlers/start.py`):** Написать асинхронные функции для обработки команд `/start` и приветствия пользователя.
* **Навигация (`keyboards/`):** Создать удобные Reply (кнопки внизу экрана) и Inline (кнопки под сообщениями) клавиатуры.

## 4. Интеграция с основным Backend API (Микросервисы rrtm)
Так как у вас уже есть микросервисная архитектура и сервис авторизации:
* Разработать модуль `api_client/client.py` для выполнения HTTP-запросов к вашему API.
* При вводе команды `/start` проверять, привязан ли Telegram ID пользователя к аккаунту в вашей системе.
* Реализовать логику авторизации (возможно, запросить номер телефона через Telegram или отправить ссылку на вход с генерацией одноразового кода).

## 5. Дополнительный функционал (Опционально)
* **Машина состояний (FSM):** Использование Finite State Machine из `aiogram` для создания многошаговых диалогов (например, заполнение формы регистрации).
* **Middlewares:** Написание промежуточных слоев для проверки прав (RBAC) или логирования всех действий прямо в Telegram.

## 6. Деплой и запуск (Deployment)
* **Контейнеризация:** Написать `Dockerfile` для упаковки бота.
* **Интеграция в инфраструктуру:** Добавить запуск бота в общий `docker-compose.yml` файл проекта.
* **Выбор способа работы:** Для небольших нагрузок использовать `Long Polling`. Для production среды (при наличии HTTPS домена) перевести бота на `Webhooks`.
